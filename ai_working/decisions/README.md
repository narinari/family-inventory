# Architecture Decision Records (ADR)

このディレクトリには、プロジェクトのアーキテクチャ決定記録を保存します。

## ADR とは

Architecture Decision Record (ADR) は、重要なアーキテクチャ決定とその理由を記録するドキュメントです。

## フォーマット

ファイル名: `NNNN-title.md` (例: `0001-use-firebase.md`)

```markdown
# ADR-NNNN: タイトル

## ステータス

Proposed | Accepted | Deprecated | Superseded by ADR-XXXX

## コンテキスト

決定が必要になった背景や課題を説明します。

## 決定

何を決定したかを明確に記述します。

## 選択肢

### 選択肢1: [名前]
- メリット: ...
- デメリット: ...

### 選択肢2: [名前]
- メリット: ...
- デメリット: ...

## 理由

なぜこの決定をしたのかを説明します。

## 影響

この決定による影響（コード変更、運用変更など）を記述します。

## 関連

- 関連するADR
- 関連するドキュメント
- 関連するイシュー
```
